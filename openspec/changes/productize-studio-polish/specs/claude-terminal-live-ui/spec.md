## ADDED Requirements

### Requirement: Claude 调用在无输出时仍提供状态解释
系统 SHALL 在 Claude 调用尚未产生终端输出时展示结构化状态提示，至少覆盖排队中、已启动但无输出、慢启动告警，以及启动前网络/域名预检失败这几类状态。

#### Scenario: 展示排队中的调用状态
- **WHEN** 当前节点轮次已经成为焦点，但关联的 executor 或 validator 调用尚未真正启动
- **THEN** 终端面板或相邻状态区展示该轮次正在排队或等待派发，而不是显示为空闲

#### Scenario: 展示慢启动告警
- **WHEN** Claude 调用已经启动但在首字节阈值内持续无输出，且尚未达到整体超时
- **THEN** 系统展示慢启动告警，并说明该调用仍在运行而不是已经失败

#### Scenario: 展示预检失败原因
- **WHEN** Claude 调用在真正启动前就因为域名解析、代理或基础连接配置问题被拒绝
- **THEN** 系统展示可操作的失败原因，即使该次调用没有产生任何终端输出

## MODIFIED Requirements

### Requirement: 终端输出实时追加并可回放
系统 SHALL 以时间顺序实时追加 Claude Code 调用输出，并在调用结束后保留完整输出以便回看与排查；当调用尚未产出任何输出时，系统 SHALL 仍保留本次调用的状态、时间戳与相关告警/失败摘要，避免终端区域表现为“什么都没发生”。

#### Scenario: 调用完成后回看输出
- **WHEN** 一次 Claude Code 调用完成（成功或失败）
- **THEN** 用户仍可在终端面板中查看完整输出内容与最终状态

#### Scenario: 无输出调用仍可回看状态摘要
- **WHEN** 一次 Claude Code 调用在无输出的情况下失败或被判定为慢启动
- **THEN** 用户仍可在终端面板或其状态摘要中查看该次调用的时间、状态和失败/告警原因
