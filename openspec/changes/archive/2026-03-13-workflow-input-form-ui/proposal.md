## Why

当前运行工作流时的输入需要手动填写 JSON，门槛高且容易出错；同时在创建工作流时无法清晰表达每个输入字段的类型与约束，导致运行侧难以提供友好的表单体验。 
## What Changes

- 在工作流定义中补充输入字段的类型与约束信息（例如 `string/number/boolean/enum/file`、默认值、枚举范围、是否必填）。 
- 在工作流创建/编辑流程中提供输入字段类型配置能力，并将其保存到工作流定义。 
- 在运行工作流时基于输入类型渲染表单式输入，文件类型字段使用手填本地路径字符串，并自动生成运行所需的 JSON；该方式也便于后续批量启动工作流。 
- 兼容已有工作流定义：缺少类型信息时提供合理默认值并允许回退到 JSON 编辑。 

## Capabilities

### New Capabilities
- `workflow-input-forms`: 运行时基于输入定义渲染表单字段，并提供本地路径字符串等类型化输入控件。 

### Modified Capabilities
- `workflow-definition`: 工作流输入字段需要具备可用于表单渲染的类型与约束信息。 
- `workflow-authoring`: 创建/编辑工作流时支持配置输入字段类型与约束，并持久化到定义中。 
- `run-operations`: 启动运行时使用表单化输入收集参数，并生成标准化的运行输入。 

## Impact

- 影响 Workflow Studio 的创建与运行界面（表单输入组件、本地路径输入、校验提示）。 
- 影响工作流定义的验证与保存逻辑（新增输入类型/约束字段）。 
- 可能涉及运行创建接口的输入校验与兼容处理（缺省类型、旧定义兼容）。
